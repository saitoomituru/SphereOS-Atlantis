## migration

### legacy 0.2.1／0.25.1系の利用者

- MAGI legacy `0.2.1`は三層座標`0.200.1`へ、Atlantis legacy `0.25.1`は`0.250.1`へ写像します
- 旧表示、path、receiptをSource Eventとして残し、履歴をrewriteしません
- CLI／Python配布表示は互換のため`0.25.1a1`のままです
- workspace IDは`sphereos-atlantis-0.25.1-forge`になります
- `note new --shelf／--kind`は固定choicesではなく`note/registry.json`を参照します
- status確認へ`python3 -m atlantis_cli status validate`を追加します
- CORN、tutorial、experienceはoffline plan／生成を既定とし、Forgeやschedulerへ自動書込みしません
- persona tutorialは既定で`proficiency=unknown`／`intent=look-around`となり、実装を開始しません
- World接続は座標の大小でなく、SemanticKernel、World Config、capability、World Visaで判定します
- 自然言語入口は`prompt-line`、従来CLIは`command-line`として分離し、どちらも真贋軸へ置きません
- LLM／provider／connectorはLLMI／Execution Envelopeであり、interfaceや権限を自動決定しません
- `python3 -m atlantis_cli interfaces`で二つの操作面を読み取り専用表示できます
- 既定Helpは`AVAILABLE-NOW`のsummaryを先に示し、全状態は`--detail all`／`capabilities`で取得します
- Python／container／runtime境界は該当operationの要求時まで遅延表示します

このcorrective deltaは起動案内と説明レジスターの修正であり、Sphere座標`0.250.1`またはlegacy配布aliasを
刻み直しません。standalone runtime、PLI runner、model inferenceを実装したというclaimも追加しません。
起点は[Issue #4](https://github.com/saitoomituru/SphereOS-Atlantis/issues/4)です。

`0.200.0`から`0.200.1`への変更は、同一World線内でOAEを遡及再配置できるsystem上の脆性を塞ぎます。
物理的タイムトラベルの主張ではなく、backup UXとは別の意味Kernel修正です。

## release gate

1. ZeroRoomLab-manifestの横断契約PRをreview・mergeする
2. Atlantisの実装PRをreviewし、全unit test、doctor、link、clean-roomを確認する
3. community testの未確認項目をIssueへ残す
4. branch-only項目をalpha-candidateへ更新する
5. maintainerがtag作成を明示承認する

現時点ではtagもGitHub Releaseも作りません。alphaで壊れた箇所は
[Issue](https://github.com/saitoomituru/SphereOS-Atlantis/issues/new)へ送ってください。

## claim boundary

この候補が提供するのは開発足場、offline管理stack、参加・review入口です。完成OS、production runner、
model runtime、edge distribution、宗派横断の教義裁定、ゲームコンテンツの正解を提供しません。
異SemanticKernel間のcross-causal runtime gatewayも未実装で、現時点は隔離判定とfixtureまでです。
